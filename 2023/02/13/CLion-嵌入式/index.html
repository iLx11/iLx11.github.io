STM32CubeMX

选择低版本的cube MX， 因为6.4.0 以上的版本不再支持以SW4STM32为导出IDE，而我们后续需要以这个选项导出工程，故在选择下载时不可以下载最新的cubeMX

下载链接：

https://www.st.com/en/development-tools/stm32cubemx.html#get-software

OpenOCD

OpenOCD是用于对STM32进行下载仿真的工具，是一个开源软件包，解压到某个目录，后面会在Clion中链接这个目录

下载地址：

https://link.zhihu.com/?target=http%3A//gnutoolchains.com/arm-eabi/openocd/

在CLion 中配置 OpenOCD

MinGW

Clion需要使用MinGW环境来配置工具链，安装方法如下：
首先去MinGW主页下载最新版本的MinGW

下载链接：

https://link.zhihu.com/?target=https%3A//osdn.net/projects/mingw/releases/

软件运行需要有 java 环境，尽量选择低版本的下载

https://www.java.com/zh-CN/download/

安装时把Basic Setup里面的组件全部勾选（也可也去掉不需要的语言编译器比如Objective-C）。

之后配置系统的环境变量，在Path环境变量里面添加一条，指向MinGW的bin文件夹

验证安装是否成功：

gcc -v

打印出版本信息则为配置成功

arm-none-eabi-gcc

下载链接：

https://developer.arm.com/downloads/-/gnu-rm

解压后将 bin 目录配置到系统环境目录

cmd 输入

arm-none-eabi-gcc -v

如果有信息输出,那就是装好了

在工程文件夹下创建 config 目录

config 目录下 新建命名为 stlink.cfg 的文件

(st-link 上传)

文件中粘贴硬件配置

前两行设置了仿真器的类型和接口，下面几行指定了Flash大小、芯片类型、下载速度等

# choose st-link/j-link/dap-link etc.
#adapter driver cmsis-dap
#transport select swd
 
source [find interface/stlink.cfg]
transport select hla_swd
 
source [find target/stm32f1x.cfg]
 
# download speed = 10MHz
adapter speed 10000
